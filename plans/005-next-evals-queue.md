## Why this matters

582 tools are catalogued; 473 are `discovery-log` — leads that dead-end because nothing selects what to evaluate next. Stars are captured at scan time then discarded; the three report-only detectors (`--overlaps`, `--savings-claims`, `--skills`) each rank a slice but feed no decision. Meanwhile the stages with the most catalogued options have the least validated guidance (Research & Discovery 1/18 validated, Outer Loop 2/40, Memory & Context 2/47 — verified 2026-07-03). This plan derives a ranked, regenerable `NEXT-EVALS.md` so "what do we evaluate next" is a standing, data-backed queue — the same derive-don't-hand-maintain philosophy as `tier-stack.py`.

## Current state

- Verdict data: `COMPARISON.md` rows `| Tool | Type | Auto | Free | Evaluated | Evidence |`; discovery-log rows are the queue's candidate pool. Row parsing is centralized in `catalog_lib.py` (commits `1a18a12`, `ea52910`) — reuse it, never re-parse with new regexes.
- Overlap pressure: `CATALOG.md` rows carry an "Overlaps with" column; `audit-evals.py --overlaps` already computes token → citing-rows counts ("the more rows cite the same token, the likelier a gap"). Locate the function via `grep -n "overlap" audit-evals.py`.

## Steps

### Step 1: Implement the ranking (fixed formula — do not redesign)

For each discovery-log row, compute:

```
score = 2*overlap_pressure + stage_gap_weight + evidence_bonus
  overlap_pressure  = number of OTHER catalog rows citing this tool in "Overlaps with" (0 if none)
  stage_gap_weight  = 10*(1 - Validated/Tools) for the row's COMPARISON stage   # 0..10, hungriest stage highest
  evidence_bonus    = 2 if Evidence is REVIEW (some homework done) else 0 (SOURCE-ONLY)
```

Emit `NEXT-EVALS.md`: a header explaining derivation + "regenerate with `python3 next-evals.py`", then the top-25 as a table `| Rank | Tool | Stage | Score | Why (pressure/gap) | Eval command |` where Eval command is `/evaluate-tool <name>`. Include markers `<!-- NEXT-EVALS:START -->`/`END` around the generated block (tier-stack convention) and a "what was dropped" line: total candidate count vs shown (no silent caps — repo convention).

**Verify**: `python3 next-evals.py` → file exists; top entries skew toward Research & Discovery / Outer Loop / Memory & Context stages; a tool with high overlap-pressure (run `python3 audit-evals.py --overlaps` and pick the most-cited token that is itself catalogued) appears in the top 10.
