## What it does

The largest Claude Code enhancement system by scale: 271 skills, 67 agents, 92 commands, 22 language rule sets, and 5 hooks. Originally called "everything-claude-code," the project now brands as ECC and ships as both a Claude Code plugin (`ecc@ecc`) and a standalone install script. Covers 12+ language ecosystems (TypeScript, Python, Go, Rust, Java, C#, Dart, Kotlin, PHP, Ruby, Perl, Swift) with language-specific coding-style, hooks, patterns, security, and testing rules. Skills span domains from API design and benchmark optimization to brand-voice and agentic-engineering.

The mechanism is a plugin that drops SKILL.md files, agent definitions, slash commands, hooks, and rules into the agent's context. Skills use progressive disclosure — frontmatter triggers activate the right skill, then the agent loads the full reference on demand. Unlike mattpocock/skills (6 opinionated skills from one developer's workflow) or addyosmani/agent-skills (11 lifecycle skills), ECC is a breadth play: cover every domain, language, and role with a consistent format.

## How we tested it

Architecture review via GitHub API: enumerated skills, agents, commands, hooks, and rules. Read 4 representative SKILL.md files for quality assessment (api-design, benchmark-optimization-loop, agentic-engineering, brand-voice). Checked the local install — two auto-generated conventions skills exist at `~/.claude/skills/everything-claude-code/` but these are the repo's own contribution patterns (generated by ECC Tools), not the plugin's user-facing skills. The actual plugin was not installed.

## What worked

- **Quality is surprisingly high for the scale.** api-design gives opinionated URL structure, status codes, pagination, and error response patterns with concrete code. benchmark-optimization-loop defines a measured optimization loop with variant tables and stopping criteria. These are not boilerplate — they encode specific engineering practices.
- **Language rule coverage is broad and practical.** 22 language-specific rule directories each with coding-style, hooks, patterns, security, and testing guidelines. Having Rust, Go, Swift, Perl alongside the usual TypeScript/Python is genuinely useful for polyglot teams.
- **Cross-harness compatibility.** Ships skill copies for Codex (`.agents/`), Cursor (`.cursor/`), Gemini (`.gemini/`), Zed (`.zed/`), and more. One install covers multiple AI coding tools.
- **Active development at scale.** 230+ contributors, v2.0.0 with Hermes operator story, GitHub App with CI integration.

## What didn't work or surprised us

- **Locally installed skills are wrong artifacts.** The `everything-claude-code` and `everything-claude-code-conventions` SKILL.md files in `~/.claude/skills/` are auto-generated contribution guides for the ECC repo itself — they describe ECC's commit conventions, not useful domain skills. Users installing via `npx skills add` get the wrong thing.
- **271 skills create a massive context surface.** Without careful activation filtering, loading too many skills simultaneously would destroy the context window. The progressive disclosure mitigates this but relies on good frontmatter triggers.
- **Hard to verify 271 skills.** Spot-checking 4 showed high quality, but the long tail of 267 others could contain filler. Community contributions at this scale are inherently uneven.
- **Relationship to ECC harness is confusing.** ECC is simultaneously a harness, a plugin, a skill collection, a rules framework, and a CI tool. The catalog entry says "plugin" but it's really a meta-system.

## Verdict

**CONDITIONAL**

Use ECC when you need broad domain coverage across multiple languages and AI coding tools (Codex, Cursor, Claude Code) in a polyglot team. The breadth — 271 skills, 22 language ecosystems, cross-harness compatibility — is unmatched. However, the locally installed skills via `npx skills add` deliver the wrong artifact (repo conventions, not user skills), so install the full plugin (`ecc@ecc`) instead. For single-language or focused workflows, curated collections like mattpocock/skills (TypeScript), trailofbits/skills (security), or web-quality-skills (web quality) deliver more targeted value with less context overhead.
