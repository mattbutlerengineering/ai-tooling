The loop

One loop, six stages

Every change travels the same path. The loop closes when what we ship teaches us what to build next.

01 Plan Scope the smallest useful change.
02 Build Write it in small, tested increments.
03 Review A second reader before merge.
04 Integrate Merge to main behind green CI.
05 Ship Release on demand, in small batches.
06 Observe Watch production; feed it back to 01.
Stage 01

Plan and scope

Work starts as a written problem, not a ticket to code.

Define the problem and the smallest change that solves it.
Agree on acceptance criteria before any code is written.
Break work into increments that ship in days, not weeks.

Stage 02

Build

Small commits, green tests, trunk always shippable.

Keep branches short-lived and diffs small enough to review.
Write the test with the change, not after it.
Leave the code clearer than you found it.

Stages 03–04

Review and integrate

Nothing reaches main without a second reader and a green build.

Review for correctness, clarity, and blast radius.
Automate the rest — format, lint, and tests run in CI.
Merge small and often; resolve conflicts while they're cheap.

Stages 05–06

Ship and observe

Releasing is routine, reversible, and watched.

Deploy in small batches; roll back faster than you roll forward.
Guard risky changes behind feature flags.
Watch the signals that matter, then feed them back to planning.
